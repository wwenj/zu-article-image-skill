# Illustration Types

类型决定图片的信息结构，只用于帮助设计自然语言 Prompt，不作为生图工具参数。

## 类型

- **infographic / 概念信息图**：表达抽象概念、核心观点、数据摘要和相关要素。明确中心概念、分区和 takeaway。
- **flowchart / 流程图**：表达步骤、生命周期和执行链路。明确顺序、输入输出、起点终点和箭头方向。
- **comparison / 对比图**：表达 Before/After、传统方案 vs 新方案或方案差异。两侧使用相同对比维度。
- **framework / 框架图**：表达模块、角色、层级、依赖、边界和数据流。禁止增加文章未提供的组件。
- **scene / 场景图**：表达观点、叙事和视觉隐喻。优先表达底层含义，避免通用装饰图。
- **timeline / 时间线**：表达阶段、演进、里程碑和趋势。明确方向、节点和关键转折。

## 自动选择提示

- 文章在解释一个概念或总结多个观点时，优先 `infographic`。
- 文章强调步骤、执行、处理链路时，优先 `flowchart`。
- 文章出现 vs、before/after、优缺点、传统/新方案时，优先 `comparison`。
- 文章讨论系统、架构、模型、边界、模块关系时，优先 `framework`。
- 文章主要是观点、叙事、隐喻或情绪表达时，才使用 `scene`。
- 文章按时间或阶段展开时，优先 `timeline`。

无法明确说明“这张图帮助读者理解什么”时，不插入标签。
