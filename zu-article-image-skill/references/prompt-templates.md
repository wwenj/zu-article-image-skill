# Prompt Templates

模板只规定 Prompt 应该覆盖的信息，不要求逐字照抄。最终必须写成标签正文里的自然语言 Prompt。

每个 Prompt 都必须包含：

- Purpose：这张图帮助读者理解什么。
- Layout：整体构图和阅读方向。
- Content：文章中的真实术语、角色、步骤、数据或关系。
- Style：画面风格和渲染规则。
- Palette：颜色语义，颜色名和 hex 值不能显示在图片里。
- Text：图片中文字必须短、清晰、匹配文章语言。
- Aspect：比例，默认 `16:9`。

## infographic

用于概念总结、知识卡片、数据摘要。

必须写清：

- Layout：网格、分区、中心辐射或上下结构。
- Zones：每个信息区展示什么。
- Labels：使用文章里的关键词、数字、指标或短句。
- Takeaway：可选，一句底部总结。

避免：无意义图标堆叠、长段文字、文章没有提到的数据。

## flowchart

用于步骤、流程、生命周期、执行链路。

必须写清：

- Layout：从左到右、自上而下或循环。
- Steps：每一步的名称和简短作用。
- Connections：箭头方向、分支、输入输出。
- Start/End：起点和终点。

避免：没有方向的箭头、步骤遗漏、把模块画成装饰物。

## comparison

用于方案差异、Before/After、传统与新方法。

必须写清：

- Layout：左右分栏或上下对照。
- Left/Right：两侧分别代表什么。
- Dimensions：两侧使用相同对比维度。
- Divider：视觉分隔方式。

避免：左右维度不一致、情绪化夸张、只画胜负不画原因。

## framework

用于架构、模型、系统边界、概念关系。

必须写清：

- Structure：分层、中心辐射、矩阵或网络。
- Nodes：关键节点和角色。
- Relationships：依赖、数据流、控制流或影响关系。
- Boundaries：系统内外、已实现和未来能力。

避免：增加文章未提供的模块、连接线过多、层级不清。

## scene

用于观点、叙事、情绪和隐喻。

必须写清：

- Focal Point：单一主视觉。
- Metaphor：表达底层观点，不机械直画文字比喻。
- Atmosphere：光线、氛围、空间关系。
- Visual Restraint：少元素、少文字、强语义。

避免：通用装饰图、照片感人物、复杂背景。

## timeline

用于历史、演进、阶段变化。

必须写清：

- Direction：水平或垂直。
- Events：阶段、时间点或里程碑。
- Markers：节点形状和当前重点。
- Progression：变化趋势和关键转折。

避免：日期缺失、节点密度过高、只有装饰线。
